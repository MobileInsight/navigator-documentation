1. Resumen​

En la transcripción se explica cómo funciona la tabla for data de hen y cómo se deben realizar las consultas para obtener información de ella. Se dan varios ejemplos de consultas y se explica cómo interpretar los resultados. 2. Temas desarrollados​

La tabla for data La tabla for data es una tabla sencilla que contiene información de las visitas realizadas a una tienda de un cliente. Cada fila de la tabla representa una visita y contiene información sobre el número de la visita, la fecha de la visita, la tienda en la que se realizó la visita, el tipo de formulario utilizado y las respuestas a las preguntas del formulario. Esta información se utiliza para analizar el rendimiento de las tiendas y para mejorar la experiencia de los clientes.

La tabla for data tiene 511 columnas, pero no todas las columnas están presentes en todas las visitas. Las columnas que están presentes en una visita dependen del tipo de formulario utilizado para realizar la visita.

El número de la visita se almacena en la columna form id. El tipo de formulario se almacena en la columna form.

La tabla for data también contiene información sobre la fecha y hora de la visita, la tienda visitada, el empleado que realizó la visita, etc.

Preguntas que se pueden hacer en una visita Las preguntas se pueden clasificar en dos tipos: preguntas estándar y preguntas específicas de cada tipo de visita.

Las preguntas estándar son aquellas que se hacen en todas las visitas, independientemente del tipo de tienda que se visite. Estas preguntas incluyen información sobre la fecha y hora de la visita, la tienda que se visitó, quién hizo la visita, etc.

Las preguntas específicas de cada tipo de visita son aquellas que se hacen solo en visitas de un tipo específico de tienda. Por ejemplo, en una visita a una tienda de electrónica, se puede preguntar si el producto que se busca está en stock, si el precio es el adecuado, etc.

Todas las preguntas se almacenan en la tabla for data. El número de la visita se almacena en el campo form id, y el tipo de pregunta se almacena en el campo column name. La respuesta a la pregunta se almacena en el campo data.

Form ID y Formit El form id es el número de la visita y el formit es el tipo de formulario que se está utilizando. Obtener el store id de una visita Para obtener el store id de una visita, se debe utilizar el campo 03 de la tabla hensenformdata. Este campo contiene el id de la tienda donde se realizó la visita.

Obtener la fecha de actualización de una visita Para obtener la fecha de actualización de una visita, se puede utilizar el siguiente query:

SELECT form_id,
       store_id,
       date,
       region_name,
       district_name,
       market_name
FROM hensen_form_data
JOIN hen_stores
ON hensen_form_data.store_id = hen_stores.store_id
WHERE form_id = 3793
AND updated date IS NOT NULL
LIMIT 100;

Este query selecciona los siguientes campos de las dos tablas:

  • form_id: El número de la visita.
  • store_id: El ID de la tienda.
  • date: La fecha de la visita.
  • region_name: El nombre de la región.
  • district_name: El nombre del distrito.
  • market_name: El nombre del mercado.

El query también utiliza el operador JOIN para unir las dos tablas por el campo store_id.

El operador WHERE se utiliza para filtrar los resultados por el campo form_id y el campo updated_date.

El campo form_id se utiliza para seleccionar solo las visitas del formulario 3793.

El campo updated_date se utiliza para seleccionar solo las visitas que han sido actualizadas.

El límite de 100 se utiliza para limitar el número de resultados que se devuelven.

Join entre dos tablas Se debe utilizar el comando select para traer los campos form id, store id, fecha, nombre de la región, nombre del distrito y nombre del market de las dos tablas. Los tres primeros campos del select salen de la tabla henenseform data y los tres últimos campos salen de la tabla hen stores. Como ya se vio en el query anterior, la columna triple0 piso 013 es la que trae el store id, donde lo consigues en el campo data de esa columna y el updated date sale de la columna triple0 piso 055, donde lo consigues en el campo data de esa columna. Todo esto se debe hacer utilizando formit que es el tipo de formulario, no form id que es el número de visita, formit 3793 y por favor utilicen un límite inclusive para las pruebas no solo para el query final.

With para hacer subqueries Ejemplo para utilizar el with para hacer subqueries:

select form_id, store_id, date, updated_date
from hensenseform_data
where form_id = 3793
limit 100;

Este query trae el form_id, store_id, date y updated_date de la tabla hensenseform_data, donde el form_id es igual a 3793 y se le pone un límite de 100.

Límite en un query El límite se utiliza para indicar el número de filas que se van a devolver en un query. Se puede utilizar en cualquier tipo de query, pero es especialmente útil cuando se trabaja con grandes cantidades de datos.

Para utilizar el límite, se utiliza la palabra clave "limit" seguida del número de filas que se desean devolver. Por ejemplo, el siguiente query devolverá las primeras 10 filas de la tabla "users":

SELECT * FROM users LIMIT 10;

El límite se puede utilizar junto con otros operadores de consulta, como where y order by. Por ejemplo, el siguiente query devolverá las primeras 10 filas de la tabla "users" que tengan el nombre "John":

SELECT * FROM users WHERE name = "John" LIMIT 10;
El límite también se puede utilizar para paginar los resultados de un query. Por ejemplo, el siguiente query devolverá las primeras 10 filas de la tabla "users", y luego se ejecutará nuevamente para devolver las siguientes 10 filas, y así sucesivamente:
SELECT * FROM users LIMIT 10 OFFSET 0;
SELECT * FROM users LIMIT 10 OFFSET 10;
SELECT * FROM users LIMIT 10 OFFSET 20;

Y así sucesivamente. Enunciado ejercicio práctico Copiar el siguiente query:

select form_id,
store_id,
data as store_id,
updated_date
from hensenseform_data
where form_id = 3793
limit 100;

Este query traerá el form_id, el store_id, la fecha y el updated_date de la tabla hensenseform_data, utilizando el form_id = 3793 como filtro. Este query se tiene que trabajar en staging y con un límite de 100 para no traer toda la data.

3. Preguntas más frecuentes​

¿Qué es una visita? Una visita es un formulario que llena la persona que va a hacer la visita.

¿Cuáles son los distintos tipos de preguntas que se pueden hacer en una visita? Hay distintos tipos de preguntas que se pueden hacer en una visita, por ejemplo:

• ¿Dónde está el producto? • ¿Cómo funciona el producto? • ¿Qué tan satisfecho está con el producto? • ¿Recomendaría el producto a un amigo? • ¿Qué podría mejorarse en el producto?

Estas son solo algunas de las preguntas que se pueden hacer en una visita, y las respuestas a estas preguntas pueden ayudar a las empresas a mejorar sus productos y servicios.

¿Cómo se guarda la información de una visita en la tabla hensenformdata? La información de una visita se guarda en la tabla hensenformdata en forma de registros. Cada registro corresponde a una pregunta y respuesta de la visita.

¿Qué es el form id? El form id es el número de la visita.

¿Qué es el formit? El formit es el tipo de formulario que se está utilizando.

¿Dónde puedo encontrar la lista de formularios que hay disponibles? Puedes encontrar la lista de formularios disponibles en la tabla forms.

¿Cómo se llama la columna que contiene el store id? La columna que contiene el store id es la columna 03.

¿Cómo se llama la columna que contiene la fecha? La columna que contiene la fecha es la columna 055.

¿Cómo obtener el store id de una visita? Para obtener el store id de una visita, se puede utilizar el siguiente query:

SELECT form_id,
       store_id,
       date,
       region_name,
       district_name,
       market_name
FROM hensenform_data
JOIN hen_stores
ON hensenform_data.store_id = hen_stores.store_id
WHERE form_id = 3793
LIMIT 100;

¿Cómo se puede consultar la información de una visita en la tabla hensenformdata? Para consultar la información de una visita en la tabla hensenformdata, se puede utilizar el siguiente query:

select form_id, store_id, date, region, district, market
from hensenformdata
join hen_stores on hensenformdata.store_id = hen_stores.store_id
where form_id = 3793
limit 100;

Este query selecciona el form_id, store_id, date, region, district y market de la tabla hensenformdata. El store_id se utiliza para unir las dos tablas, y el form_id se utiliza para seleccionar solo las filas que corresponden a la visita con el id 3793. El límite de 100 se utiliza para evitar que se devuelvan todas las filas de la tabla.

¿Cómo se puede utilizar la tabla hensenformdata para analizar la información de las visitas? La tabla hensenformdata se puede utilizar para analizar la información de las visitas mediante el uso de consultas. Las consultas permiten extraer información específica de la tabla, como el número de visitas, el tipo de formulario utilizado, la fecha y la hora de la visita, la tienda visitada, etc. Las consultas también se pueden utilizar para realizar análisis más complejos, como la comparación de datos entre diferentes visitas o la identificación de tendencias.

¿Cómo obtener la fecha de actualización de una visita? Para obtener la fecha de actualización de una visita, se puede utilizar el siguiente query:

SELECT form_id,
  store_id,
  date,
  region_name,
  district_name,
  market_name
FROM hensen_form_data
JOIN hen_stores ON hensen_form_data.store_id = hen_stores.store_id
WHERE form_id = 3793
AND updated_date IS NOT NULL
LIMIT 100;

Este query obtiene el form_id, store_id, date, region_name, district_name y market_name de las tablashensen_form_data y hen_stores. El form_id y store_id se obtienen de la tabla hensen_form_data, mientras que la fecha, la región, el distrito y el mercado se obtienen de la tabla hen_stores. El query también utiliza el operador AND para filtrar los resultados por el form_id y el updated_date. El límite de 100 se utiliza para limitar el número de resultados que se devuelven.

¿Qué es un join? Un join es una operación que combina dos o más tablas de una base de datos en una sola tabla. Esto permite obtener información de ambas tablas a la vez.

¿Cómo se hace un join? Para hacer un join, se utilizan las palabras clave "join" y "on". La palabra "join" indica que se va a realizar un join, y la palabra "on" indica el campo o campos que se utilizarán para unir las dos tablas.

¿Cuáles son los diferentes tipos de joins?

Los diferentes tipos de joins son: • Inner join: El inner join devuelve todos los registros de la tabla izquierda que coinciden con los registros de la tabla derecha. • Left outer join: El left outer join devuelve todos los registros de la tabla izquierda, incluso si no hay coincidencias en la tabla derecha. • Right outer join: El right outer join devuelve todos los registros de la tabla derecha, incluso si no hay coincidencias en la tabla izquierda. • Full outer join: El full outer join devuelve todos los registros de ambas tablas, incluso si no hay coincidencias en ninguna de las tablas.

¿Cuál es el join más utilizado? El join más utilizado es el inner join. Esto se debe a que el inner join solo devuelve los registros que coinciden en ambas tablas, lo que evita que se devuelvan registros duplicados.

¿Cómo puedo hacer un join en PostgreSQL? • Para hacer un join en PostgreSQL, se utilizan las siguientes instrucciones:

•   SELECT * FROM table1 JOIN table2 ON table1.column1 = table2.column2;
•   SELECT * FROM table1 LEFT JOIN table2 ON table1.column1 = table2.column2;
•   SELECT * FROM table1 RIGHT JOIN table2 ON table1.column1 = table2.column2;
•   SELECT * FROM table1 FULL JOIN table2 ON table1.column1 = table2.column2;

¿Cómo se utiliza el with para hacer subqueries? El with se utiliza para crear un alias para una consulta y luego utilizar ese alias en otra consulta. Esto permite crear consultas más complejas y eficientes.

¿Cuáles son las ventajas de utilizar el with para hacer subqueries? Las ventajas de utilizar el with para hacer subqueries son: • Permite crear consultas más complejas y eficientes. • Reduce la cantidad de código que se necesita escribir. • Facilita la comprensión de las consultas.

¿Cuáles son los pasos para utilizar el with para hacer subqueries? Los pasos para utilizar el with para hacer subqueries son:

  1. Crear un alias para la consulta que se desea utilizar.
  2. Utilizar el alias en la otra consulta.

¿Cuál es un ejemplo de una consulta con with? El siguiente es un ejemplo de una consulta con with:

WITH my_table AS (
  SELECT * FROM table_name
)
SELECT * FROM my_table

En este ejemplo, la consulta con with crea un alias llamado my_table para la consulta que se realiza en la primera línea. Luego, la consulta con with utiliza el alias my_table en la segunda línea para seleccionar todos los registros de la tabla.

¿Qué es el límite en un query? El límite en un query es una instrucción que permite establecer el número máximo de filas que se devolverán como resultado del query.

¿Cómo se utiliza el límite en un query? Para utilizar el límite en un query, se utiliza la palabra clave "limit" seguida del número de filas que se desean devolver. Por ejemplo, el siguiente query devolverá las primeras 10 filas de la tabla "users":

SELECT * FROM users LIMIT 10;

¿Por qué es importante utilizar el límite en un query? El límite es importante en un query porque puede ayudar a mejorar el rendimiento de la consulta. Al limitar el número de filas que se devuelven, se reduce el tiempo que tarda la consulta en ejecutarse.

¿Cuáles son las ventajas de utilizar el límite en un query? Las ventajas de utilizar el límite en un query son: • Mejora el rendimiento de la consulta. • Reduce el uso de recursos del servidor. • Evita que se devuelvan demasiados datos, lo que puede dificultar la interpretación de los resultados.

¿Cuáles son las desventajas de utilizar el límite en un query? Las desventajas de utilizar el límite en un query son: • Si se establece un límite demasiado pequeño, es posible que se pierdan datos importantes. • Si se utiliza el límite en un query que se ejecuta con frecuencia, puede afectar al rendimiento del servidor.
