Introducción a Navigator

Introducción a Navigator​

Navigator es una aplicación web que permite visualizar y analizar datos mediante dashboards y widgets. En la primera hora de la reunión de hoy vamos a explicar paso a paso y con ejemplos como gestionar la base de datos de Navigator.

La base de datos de Navigator está compuesta por varias tablas, las principales son stores, store details, bw stores y store types.

En la tabla stores se encuentra la información de las tiendas, como su ubicación, nombre, tipo de tienda, etc.

En la tabla store details se encuentra información más específica de las tiendas, como el número de visitas que han recibido, los objetivos que tienen, etc.

La tabla bw stores es una vista que contiene solo las tiendas activas.

La tabla store types contiene los distintos tipos de tiendas que existen.

A lo largo de la reunión se explicará cómo utilizar estas tablas para realizar consultas y obtener información de las tiendas.

Estructura de las tablas de Navigator

La estructura de las tablas de Navigator es la siguiente:

  • Stores: Es la tabla principal que contiene información sobre las tiendas, como su ubicación, tipo, etc.

  • Store details: Es una tabla que contiene información específica sobre las tiendas, como el número de visitas que han recibido, etc.

  • BW stores: Es una vista que contiene solo las tiendas activas para un programa determinado.

Para realizar consultas en Navigator, es importante conocer la estructura de las tablas y los distintos tipos de joins que se pueden utilizar.

Consultas básicas en Navigator​

Para hacer una consulta en Navigator debemos conocer las tablas que tenemos disponibles y los campos que contienen.

Las tablas básicas son stores y store details.

En stores tenemos información sobre las tiendas como su ubicación, tipo, etc.

En store details tenemos información más específica sobre las tiendas que el cliente quiere trabajar.

Para hacer una consulta debemos utilizar el comando SELECT seguido de los campos que queremos ver.

Podemos utilizar condiciones para filtrar los resultados de la consulta.

Podemos utilizar joins para unir dos o más tablas.

Los joins se pueden hacer de dos formas: inner join y left join.

El inner join solo trae los registros que coinciden en las dos tablas.

El left join trae todos los registros de la tabla izquierda y los que coinciden en la tabla derecha.

Podemos utilizar funciones para transformar los datos de los campos.

Podemos utilizar operadores lógicos para combinar condiciones.

Podemos utilizar el comando ORDER BY para ordenar los resultados de la consulta.

Podemos utilizar el comando LIMIT para limitar el número de resultados de la consulta.

Ejercicios prácticos Este ejercicio con consultas en Navigator es cómo convertir un campo fecha hora a solo fecha. Para ello, se debe colocar el campo entre comillas dobles y luego dos puntos y la palabra date. De esta forma, se le está indicando a Postgres que utilice ese campo pero que lo utilice solo tipo fecha, obviando la hora, minuto y segundo.

Cómo hacer un query en Redshift El primer tema tratado en la transcripción de cómo hacer un query en Redshift es cómo hacer un select en Redshift. Para ello, se debe seleccionar la tabla que se quiere consultar, los campos que se quieren mostrar y las condiciones que se quieren aplicar. A continuación, se muestra un ejemplo de un select en Redshift:

SELECT region_name, count(*)
FROM stores
WHERE region_name = 'Northeast'
GROUP BY region_name;

Este query selecciona el campo region_name y el campo count(*) de la tabla stores. Además, aplica la condición region_name = 'Northeast', que filtra los resultados para que solo se muestren los registros de la región Northeast. Finalmente, el query agrupa los resultados por el campo region_name.

Cómo hacer un join, un outer join y un self join en Redshift

Un join es una operación que permite combinar dos o más tablas en una sola tabla.

Un outer join es un tipo de join que devuelve todas las filas de la tabla de la izquierda, incluso si no hay una fila coincidente en la tabla de la derecha.

Un self join es un tipo de join que combina una tabla con ella misma.

Se mostraron ejemplos de cómo utilizar los joins, los outer joins y los self joins en Redshift.

Cómo hacer un case en Redshift. Se explica cómo usar el case para cambiar el valor de un campo, para hacer cálculos condicionales y para crear variables

El case es una instrucción en Redshift que se utiliza para cambiar el valor de un campo, para hacer cálculos condicionales y para crear variables.

Para cambiar el valor de un campo, se utiliza la siguiente sintaxis:

CASE WHEN condición THEN valor_si_verdadero ELSE valor_si_falso END

Por ejemplo, para cambiar el valor del campo status a activo si el valor del campo is_active es true, y a inactivo si el valor del campo is_active es false, se puede utilizar la siguiente instrucción:

CASE WHEN is_active THEN 'activo' ELSE 'inactivo' END

Para hacer cálculos condicionales, se utiliza la siguiente sintaxis:

CASE WHEN condición THEN cálculo_si_verdadero ELSE cálculo_si_falso END

Por ejemplo, para calcular el precio de un producto si el producto está en oferta, y el precio original del producto si el producto no está en oferta, se puede utilizar la siguiente instrucción:

CASE WHEN is_on_sale THEN price * 0.9 ELSE price END

Cómo hacer un having en Redshift.

Se explica cómo usar el having para filtrar los resultados de un query

El having es una instrucción dentro de eseql que se escribe tal cual suena y se utiliza para filtrar los resultados de un query. Se coloca al final del query, después del group by y antes del order by.

La sintaxis del having es la siguiente:

having <condición>

Donde <condición> es una expresión booleana que se utiliza para filtrar los resultados del query.

Por ejemplo, para filtrar los resultados de un query por el porcentaje de tiendas focus, se puede utilizar la siguiente condición:

having percentage_of_focus_stores < 50

Cómo hacer un group by en Redshift

El group by es una instrucción en Redshift que se utiliza para agrupar los datos y calcular estadísticas.

Para utilizar el group by, primero se debe seleccionar los campos que se desean agrupar. Luego, se debe utilizar la instrucción group by seguida de los campos que se desean agrupar.

Por ejemplo, para agrupar los datos por región y calcular el número de tiendas por región, se puede utilizar el siguiente query:

select region, count(*) as stores
from bw_stores
group by region
El resultado de este query sería una tabla con dos columnas: región y número de tiendas

Preguntas más frecuentes

¿Qué es una vista en Navigator?
 Una vista es una tabla virtual que se crea a partir de una o más tablas existentes. Las vistas se utilizan para organizar y presentar los datos de una manera más conveniente para el usuario.

¿Cuáles son las tablas más importantes en Navigator?
 Las tablas más importantes en Navigator son las tablas de tiendas, store details y activities. Las tablas de tiendas contienen información sobre las tiendas, como su ubicación, tipo y estado. Las tablas de store details contienen información adicional sobre las tiendas, como sus atributos específicos del programa. Las tablas de activities contienen información sobre las actividades que se han realizado en las tiendas, como las visitas y las ventas.

¿Cómo gestionar la base de datos de Navigator?
 Para gestionar la base de datos de Navigator, es necesario tener conocimientos de SQL.

¿Cómo puedo consultar los datos en Navigator?
 Para consultar los datos en Navigator, puede utilizar la herramienta de consultas. La herramienta de consultas le permite seleccionar las tablas que desea consultar, así como los campos que desea mostrar en los resultados de la consulta.

¿Cómo puedo crear una vista en Navigator?
 Para crear una vista en Navigator, debe utilizar la herramienta de creación de vistas. La herramienta de creación de vistas le permite seleccionar las tablas que desea incluir en la vista, así como los campos que desea mostrar en la vista.

¿Cómo puedo modificar una vista en Navigator?
 Para modificar una vista en Navigator, debes utilizar la herramienta de modificación de vistas. La herramienta de modificación de vistas te permite cambiar los campos que se incluyen en la vista, así como los filtros que se aplican a la vista.

¿Cómo puedo eliminar una vista en Navigator?
 Para eliminar una vista en Navigator, debes utilizar la herramienta de eliminación de vistas. La herramienta de eliminación de vistas te permite eliminar una vista de forma permanente.

¿Cómo puedo exportar los datos de Navigator?
 Para exportar los datos de Navigator, puede utilizar la herramienta de exportación de datos. La herramienta de exportación de datos le permite exportar los datos a un archivo CSV o XLSX.

¿Cómo puedo ver todas las tiendas activas y que sean covered en heisen?
 Para ver todas las tiendas activas y que sean covered en heisen, se puede utilizar el siguiente query:

SELECT *
FROM stores
WHERE cover = true
AND active = true

¿Cómo puedo ver las tiendas que tengan visitas no hemos llegado a citas las que sean de tipo bby que hayan sido aperturadas ahorita en el 2023?
 Para ver las tiendas que tengan visitas no hemos llegado a citas las que sean de tipo bby que hayan sido aperturadas ahorita en el 2023, se puede utilizar el siguiente query:

SELECT *
FROM stores
WHERE created_at >= '2023-01-01'
AND retail_slock = 'bestbuy'

¿Cómo convierto un campo fecha hora solo a fecha en un query?
 Para convertir un campo fecha hora solo a fecha en un query, se debe utilizar el operador ::date. Por ejemplo, si el campo se llama created_at, se puede utilizar el siguiente código:

SELECT *
FROM stores
WHERE created_at::date >= '2023-01-01';

¿Cómo puedo saber qué tiendas son de tipo bby?
 Para saber qué tiendas son de tipo bby, se puede utilizar el campo retail_slock. Por ejemplo, el siguiente código devolverá todas las tiendas de tipo bby:

SELECT *
FROM stores
WHERE retail_slock = 'bestbuy';

¿Cómo puedo hacer un select en Redshift?
 Para hacer un select en Redshift, debes usar el siguiente formato:

SELECT <campos> FROM <tabla> WHERE <condición>

Por ejemplo, para seleccionar todos los campos de la tabla "stores", puedes usar el siguiente comando:

SELECT * FROM stores

¿Cómo puedo hacer un join en Redshift?
 Para hacer un join en Redshift, debes usar el siguiente formato:

SELECT <campos> FROM <tabla1> JOIN <tabla2> ON <condición>

Por ejemplo, para hacer un join entre las tablas "stores" y "products", puedes usar el siguiente comando:

SELECT * FROM stores JOIN products ON stores.store_id = products.store_id

¿Cómo puedo hacer un group by en Redshift?
 Para hacer un group by en Redshift, debes usar el siguiente formato:

SELECT <campos> FROM <tabla> GROUP BY <campo>

Por ejemplo, para agrupar los datos de la tabla "stores" por "region", puedes usar el siguiente comando:

SELECT region, count(*) FROM stores GROUP BY region

¿Cómo puedo hacer un having en Redshift?
 Para hacer un having en Redshift, debes usar el siguiente formato:

SELECT <campos> FROM <tabla> GROUP BY <campo> HAVING <condición>

Por ejemplo, para seleccionar todas las regiones que tienen más de 10 tiendas, puedes usar el siguiente comando:

SELECT region, count(*) FROM stores GROUP BY region HAVING count(*) > 10
